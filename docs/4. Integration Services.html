Why Integration?

The Challenge

  • Disparate systems, apps & data sources
  • Cloud, on-premises, SaaS & partner systems
  • Different protocols, formats & APIs
  • Need for real-time & reliable communication

The Goal

  • Seamless connectivity across boundaries
  • Loosely coupled, composable architectures
  • Automate business processes end-to-end
  • Expose & manage APIs securely
Modern enterprises have dozens or hundreds of systems that need to talk to each other. Integration is about connecting these systems reliably, securely, and efficiently ‚Äî whether they're in the cloud, on-prem, or third-party SaaS platforms.

Azure Integration Services

API Management

The front door for your APIs

API Management

What Is It?

  • Full lifecycle API management platform
  • API Gateway, Developer Portal & Management Plane
  • Unified entry point for internal & external consumers

Key Capabilities

  • Security ‚Äî OAuth, JWT, certificates, IP filtering
  • Transformation ‚Äî request/response rewriting
  • Rate limiting & throttling policies
  • Analytics & developer onboarding portal
APIM has three main components: the Gateway (processes API calls, applies policies), the Azure portal management plane (define APIs, products, policies), and the Developer Portal (self-service API discovery and onboarding for consumers). It supports REST, SOAP, WebSocket, and GraphQL APIs.

Logic Apps

Visual workflow automation

Logic Apps

What Is It?

  • Cloud-based workflow automation platform
  • Visual designer ‚Äî low-code / no-code
  • 1400+ pre-built connectors
  • Connects SaaS, on-prem & custom systems

Common Scenarios

  • B2B integration (EDI, AS2, X12)
  • SaaS orchestration (O365, Dynamics, SAP)
  • Data transformation & routing

Logic Apps is the workflow engine of Azure Integration Services. It lets you design multi-step integration workflows visually. The massive connector ecosystem means you can connect to almost anything out of the box ‚Äî from Salesforce and SAP to Twitter and file systems. For B2B scenarios, Integration Accounts add EDI, schema validation, and partner management.

Standard Logic Apps run on dedicated compute (App Service plan) and support VNet integration, making them suitable for enterprise scenarios requiring network isolation. Standard also supports stateless workflows for high-throughput, low-latency scenarios.

Service Bus

Enterprise-grade messaging

Service Bus

Queues

  • Point-to-point messaging
  • Competing consumers pattern
  • FIFO with sessions
  • Dead-letter queue for failures

Topics & Subscriptions

  • Publish/subscribe pattern
  • Multiple independent subscribers
  • SQL & correlation filters
  • Fan-out to different consumers
Service Bus provides two messaging primitives: Queues for point-to-point and Topics for pub/sub. Queues support competing consumers ‚Äî multiple receivers processing messages in parallel. Topics let you broadcast to multiple subscriptions, each with filters so subscribers only receive relevant messages. Both support advanced features like sessions, dead-lettering, scheduled delivery, and duplicate detection.

These features make Service Bus the right choice for mission-critical enterprise messaging. Peek-lock ensures messages aren't lost ‚Äî the consumer must acknowledge processing before the message is removed. Sessions provide FIFO ordering within a session group. Dead-lettering captures poison messages. Auto-forwarding enables chaining entities for complex routing scenarios.

Event Grid

Reactive, event-driven architectures

Event Grid

What Is It?

  • Fully managed event routing service
  • Push-based delivery to handlers
  • First-class Azure service integration
  • Pay per event ‚Äî no idle cost

Key Concepts

  • Events ‚Äî what happened
  • Sources ‚Äî where it happened (Blob, custom, etc.)
  • Topics ‚Äî endpoint for ingesting events
  • Subscriptions ‚Äî route events to handlers
Event Grid is purpose-built for event-driven architectures. Unlike Service Bus (which is pull-based), Event Grid pushes events to subscribers via HTTP webhooks, Functions, or other supported handlers. It has deep integration with Azure services ‚Äî when a blob is created, a resource changes, or an IoT device sends data, Event Grid can route those events with minimal configuration and near real-time latency.

Event Grid Ecosystem

Event Sources

  • Azure Blob Storage
  • Resource Groups & Subscriptions
  • IoT Hub, Service Bus, Event Hubs
  • Custom topics (your applications)

Event Handlers

  • Azure Functions
  • Logic Apps
  • Webhooks (any HTTP endpoint)
  • Queues, Service Bus, Event Hubs
Event Grid connects dozens of Azure and third-party event sources to event handlers. Custom topics let you publish events from your own apps. System topics automatically capture events from Azure resources. Handlers can be serverless functions, workflows, webhooks, or other messaging services. Filtering on event type, subject, and data fields ensures handlers only receive relevant events.

Better Together

  • APIM exposes backend APIs and routes requests to Logic Apps, Functions, or direct services
  • Logic Apps orchestrates multi-step workflows, using Service Bus for reliable messaging
  • Service Bus decouples producers and consumers with guaranteed delivery
  • Event Grid reacts to Azure and custom events, triggering Functions and Logic Apps
The real power of Azure Integration Services is using them together. A typical enterprise architecture might have APIM at the edge receiving API calls, Logic Apps orchestrating a business process, Service Bus providing reliable async communication between microservices, and Event Grid triggering reactive workflows when resources change. Each service has a distinct role, but they compose naturally.

Example: Order Processing

This example shows a typical order processing flow. The client submits an order through API Management. APIM routes to a Logic App that orchestrates the workflow. The Logic App sends an inventory update to Service Bus (reliable, decoupled processing). An event is published to Event Grid for notifications. Downstream Function Apps handle inventory updates and sending customer notifications independently.

All Azure Integration Services support Managed Identity for passwordless authentication, VNet integration and private endpoints for network isolation, and Azure Monitor for observability. APIM adds OAuth/JWT policies. Service Bus and Event Grid support RBAC and SAS for access control. Logic Apps Standard can run inside an ASE for full network isolation.
